# 榔头「AI + UGC 互动答题 / 知识闯关」MVP 验收清单

> **文档身份**：MVP 阶段交付验收的"执行清单"（Contract for Delivery · Acceptance Checklist）
> **版本**：v1.0
> **出具日期**：2026-06-19
> **输入依据**：`requirements-quiz-mvp.md`、`construction-plan-quiz-mvp.md`、9 微服务代码库
> **读者**：CEO、CTO、产品总监、QA 负责人、架构师、前端/移动端负责人
> **性质**：本清单是 MVP 上线前必须"逐项勾选通过"的一票否决制文件；任何一项未通过即不允许上线。

---

## 〇、验收流程总则

1. **验收前置条件**：
   - 代码合入 `main` 分支并通过全部 CI（单元测试 + 集成测试 + 静态扫描）
   - 预发环境（Staging）完成部署，所有依赖服务正常
   - AI Service 接入生产级模型，非沙箱 mock
2. **验收参与角色**：
   - 产品负责人（主持）+ QA（记录）+ 架构师（技术核验）+ CEO（一票否决）
3. **验收结论**：
   - 🟢 **通过**：所有 MUST 项通过 + 95% SHOULD 项通过
   - 🟡 **有条件通过**：MUST 项 100% 通过，SHOULD 项 < 95% 但已登记整改计划
   - 🔴 **不通过**：任一 MUST 项不通过，直接打回
4. **验收一票否决**：
   - V1：AI 生成题目出现"正确答案 ≥ 2 个"（答案本身错误）
   - V2：未成年人账号通过任何路径进入答题玩法
   - V3：付费续命成功但 GameSession 未更新分数（资金事故）
   - V4：排行榜数据 3 分钟内不同步到好友侧
   - V5：题目接口响应出现用户笔记原文中的敏感信息

## 3. 安全验收
