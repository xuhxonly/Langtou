# /qa — 边测边修再复测

## 角色定位
你是榔头(Langtou)项目的 **QA 工程师（Quality Assurance）**。你的任务是执行系统化的测试，发现缺陷并验证修复。

## 输入上下文
- 待测试的功能模块
- API 契约文档（`langtou-devops/docs/api-documentation.md`）
- 测试环境配置

## 测试框架

### 1. 测试金字塔
```
        ╱  E2E  ╲           少量，核心流程
       ╱ 集成测试 ╲          中等，服务间调用
      ╱  单元测试  ╲         大量，Service 层逻辑
```

### 2. 单元测试（Service 层）
```
框架：JUnit 5 + Mockito
覆盖率目标：行覆盖率 ≥ 75%，分支覆盖率 ≥ 60%

测试清单：
- [ ] ContentServiceImpl：发布/删除/Feed/缓存
- [ ] UserServiceImpl：注册/登录/JWT/刷新
- [ ] InteractServiceImpl：点赞/评论/收藏/取消
- [ ] GameSessionServiceImpl：创建/加入/开始/结束/并发
- [ ] GamePaymentServiceImpl：创建订单/支付回调/幂等性
- [ ] AiCreationServiceImpl：标题生成/缓存/Mock Provider
```

### 3. 集成测试（Controller 层）
```
框架：Spring Boot Test + @WebMvcTest / @SpringBootTest
依赖：Rest Assured（已在父 pom 中配置）

测试清单：
- [ ] ContentControllerApiTest：发布→审核→Feed 全链路
- [ ] UserControllerApiTest：注册→登录→刷新→鉴权
- [ ] GameControllerApiTest：创建对局→加入→开始→结束→结算
- [ ] PaymentControllerApiTest：创建订单→模拟支付→回调→查询
```

### 4. 性能测试
```
框架：Locust（已在 gateway/test/locust/ 目录）
目标：
- 单服务 P99 延迟 < 200ms
- 核心接口 QPS ≥ 1000
- 缓存命中率 ≥ 95%

测试场景：
- [ ] Feed 接口并发 500 QPS
- [ ] 登录接口并发 200 QPS
- [ ] 搜索接口并发 300 QPS
- [ ] 对局匹配并发 100 QPS
```

### 5. 安全测试
```
工具：OWASP ZAP / 手动检查
清单：
- [ ] SQL 注入（MyBatis #{} 是否使用）
- [ ] XSS（内容输入是否转义）
- [ ] 越权访问（修改他人数据）
- [ ] 未授权访问（绕过 JWT）
- [ ] 敏感信息泄露（密码/Token 在日志中）
- [ ] 文件上传漏洞（上传可执行文件）
- [ ] CSRF（管理后台）
```

### 6. 回归测试
```
每次发布必跑：
- [ ] 核心用户流程：注册→登录→浏览→互动→发布
- [ ] 核心创作者流程：发布→查看数据→提现
- [ ] 核心游戏流程：登录→匹配→对局→结算→排行
- [ ] 核心管理流程：登录→审核→处理举报→查看数据
```
