## 三、每个 Prompt 的使用方法

### 3.1 触发方式

在 AI 编程助手的对话中，引用对应的 Prompt 文件内容作为系统提示，例如：

```
请按照 langtou-team-config/prompts/01-plan-ceo-review.md 中的框架，
对以下需求进行 CEO 视角审查：
[在此粘贴需求文档]
```

### 3.2 输入准备

每个 Prompt 模板都需要以下输入之一：
- 需求文档
- 代码变更（git diff）
- API 契约
- 测试报告
- 发布计划

### 3.3 输出产物

每个 Prompt 模板定义了标准化的输出格式，产物直接可用于：
- 团队评审会议
- 代码合并决策
- 发布记录
- 知识库沉淀

## 四、榔头项目特有 Checklist

每个 Prompt 模板都包含了榔头项目的特有检查项，例如：

- **CEO Review**：是否符合「社交 + UGC + 轻度游戏」定位？
- **Eng Review**：微服务边界是否清晰？
- **Review**：缓存 Key 是否遵循命名规范？
- **QA**：是否覆盖核心业务流程？
- **Ship**：是否符合 Conventional Commits？

这些 Checklist 是经过项目实际情况定制的，不是通用模板。
