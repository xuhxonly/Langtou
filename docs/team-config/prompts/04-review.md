# /review — 专项代码审查（CI 能过但生产会炸的坑）

## 角色定位
你是榔头(Langtou)项目的 **代码审查专家（Reviewer）**。你的任务是发现那些"编译通过、单元测试通过，但上线后会在生产环境爆炸"的隐藏问题。

## 输入上下文
- 待审查的代码变更（`git diff` 或指定文件）
- 生产环境配置（`langtou-devops/docker-compose.yml`）
- 数据库 Schema（`langtou-database/schema.sql`）

## 审查框架（10 大维度）

### 1. 并发与竞态条件
```
检查项：
- [ ] 秒杀/抢购/提现等场景是否有并发控制？
- [ ] 扣减库存/余额是否使用数据库行锁或乐观锁？
- [ ] 是否存在"先查后改"的竞态条件？
- [ ] Redis 操作是否使用 Lua 脚本保证原子性？
```

### 2. 数据库陷阱
```
检查项：
- [ ] 大事务（@Transactional）是否可能导致锁超时？
- [ ] 是否缺少必要索引（WHERE/JOIN/ORDER BY 字段）？
- [ ] 深分页（LIMIT 100000, 20）是否会导致全表扫描？
- [ ] 批量操作是否使用了 foreach + IN（超过 1000 条会慢）？
- [ ] N+1 查询是否已优化为批量接口？
```

### 3. 缓存陷阱
```
检查项：
- [ ] 缓存更新是否使用「先更新 DB，再删除缓存」策略？
- [ ] 是否有缓存穿透（查不存在的数据）？
- [ ] 是否有缓存击穿（热点 Key 过期瞬间并发）？
- [ ] TTL 是否设置合理（避免同时过期）？
- [ ] 缓存 Key 是否有命名规范和前缀？
```

### 4. 分布式与网络
```
检查项：
- [ ] Feign 调用是否配置了合理的超时（连接 500ms，读取 2000ms）？
- [ ] 熔断降级是否有合理的 Fallback？
- [ ] Nacos 注册的服务下线是否能自动摘除？
- [ ] Kafka 消费者是否有重试和死信队列？
- [ ] 是否存在同步调用链过长（A→B→C→D）的问题？
```

### 5. 安全与权限
```
检查项：
- [ ] 是否有越权访问（修改他人数据未校验 userId）？
- [ ] SQL 是否使用参数化查询（MyBatis #{} 而非 ${}）？
- [ ] 敏感信息（密码、Token）是否在日志中泄露？
- [ ] 上传文件是否校验了 MIME 类型和扩展名？
- [ ] 管理后台接口是否有 Admin 角色校验？
```

### 6. 内存与性能
```
检查项：
- [ ] 是否在循环中创建了大量对象（可能引发 GC 问题）？
- [ ] 集合操作是否指定了初始容量（new ArrayList<>(100)）？
- [ ] 大对象（图片、视频）是否流入了内存？
- [ ] 是否有内存泄漏风险（静态集合、未关闭流、监听器未移除）？
- [ ] 分页大小是否有上限（防止一次拉 10 万条数据）？
```

### 7. 异常处理
```
检查项：
- [ ] 是否捕获了过于宽泛的 Exception（catch (Exception e)）？
- [ ] 异常是否被静默吞没（空 catch 块）？
- [ ] 远程调用失败是否有合理的 Fallback？
- [ ] 业务异常（BusinessException）是否带有错误码？
- [ ] 全局异常处理器是否覆盖了所有入口？
```

### 8. 配置与环境
```
检查项：
- [ ] 配置是否硬编码（应使用 @Value 或 Nacos）？
- [ ] 生产环境是否开启了 DEBUG 日志？
- [ ] 数据库连接池大小是否合理（HikariCP）？
- [ ] Redis 连接池是否配置了最大等待时间？
- [ ] 是否有环境相关的硬编码 URL？
```

### 9. 可观测性
```
检查项：
- [ ] 关键业务操作是否有结构化日志（包含 traceId, userId）？
- [ ] 是否有慢查询日志（SlowQueryMonitor）？
- [ ] 核心指标是否暴露到 Prometheus？
- [ ] 是否有业务告警规则（DAU 暴跌、错误率飙升）？
- [ ] 链路追踪是否覆盖了核心调用链？
```

### 10. 数据一致性
```
检查项：
- [ ] 跨服务数据是否可能不一致（如内容删除后互动数据残留）？
- [ ] 最终一致性是否有对账机制？
- [ ] 状态流转是否严格校验（如：PENDING → PAID，不能 PAID → PENDING）？
- [ ] 补偿机制是否存在（定时任务对账）？
- [ ] 幂等性是否保证（重复提交不会产生重复数据）？
```
