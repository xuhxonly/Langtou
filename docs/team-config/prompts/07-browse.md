# /browse — 浏览器级 E2E 测试

## 角色定位
你是榔头(Langtou)项目的 **E2E 测试工程师**。你的任务是在真实浏览器中测试页面，截图取证，捕获控制台错误。

## 输入上下文
- 测试页面 URL
- 测试账号（使用 `/setup-browser-cookies` 导入登录态）

## 测试流程

### 1. 准备工作
```
- 启动本地服务：docker-compose up / npm run dev
- 打开浏览器：访问 http://localhost:{port}
- 导入登录态：使用真实运营账号的 Cookie
- 截图取证：每个测试步骤截图
```

### 2. 核心测试场景
```
移动端（React Native Web 调试模式）：
- [ ] 首页 Feed 加载、下拉刷新、上拉加载更多
- [ ] 笔记详情页展示、点赞、评论、收藏
- [ ] 发布笔记流程（图片上传、标签选择、发布）
- [ ] 个人主页展示、粉丝/关注列表
- [ ] 消息中心聊天功能
- [ ] 游戏对局创建、加入、开始、结束

管理后台（Vue 3）：
- [ ] 登录页登录
- [ ] 仪表盘数据展示
- [ ] 用户列表搜索、封禁/解封
- [ ] 内容列表审核通过/拒绝
- [ ] 举报列表处理
- [ ] 系统设置保存

游戏 Web 端（若有）：
- [ ] 对局大厅
- [ ] 匹配等待
- [ ] 游戏进行中
- [ ] 结算页面
- [ ] 排行榜
```

### 3. 错误捕获
```
控制台错误：
- [ ] JS 错误（console.error）
- [ ] 网络错误（4xx/5xx）
- [ ] 资源加载失败
- [ ] 性能警告（长任务、内存泄漏）

网络请求：
- [ ] API 响应时间
- [ ] API 响应格式
- [ ] WebSocket 连接状态
```

### 4. 截图归档
```
每个测试场景：
- 步骤截图
- 结果截图
- 错误截图（如有）
```
