## 一、各平台运营策略

### 1.1 平台矩阵总览

| 平台 | 定位 | 优先级 | 目标人群 | 核心目标 |

## 三、KOL/KOC合作策略

### 3.1 达人分级体系

| 级别 | 粉丝量 | 合作模式 | 单次费用 | 月度合作量 | 核心目标 |
|------|--------|----------|----------|------------|----------|
| **S级** | 500万+ | 年度合作/品牌大使 | ¥20-50万 | 1-2人 | 品牌背书+大规模曝光 |
| **A级** | 100-500万 | 季度合作/ campaign合作 | ¥5-20万 | 3-5人 | 垂类渗透+话题引爆 |
| **B级** | 10-100万 | 月度合作/ 内容定制 | ¥1-5万 | 10-15人 | 内容供给+精准转化 |
| **C级** | 1-10万 | 月度合作/ 产品体验 | ¥0.3-1万 | 30-50人 | 口碑铺量+真实感 |
| **KOC** | <1万 | 产品置换/ 佣金分成 | 产品/佣金 | 100-200人 | 素人口碑+社区氛围 |

### 3.2 达人合作预算（首季度）

| 级别 | 人数 | 人均费用 | 季度预算 | 预期产出 |
|------|------|----------|----------|----------|
| S级 | 2人 | ¥30万 | ¥60万 | 品牌背书+5000万曝光 |
| A级 | 8人 | ¥8万 | ¥64万 | 话题引爆+3000万曝光 |
| B级 | 30人 | ¥2万 | ¥60万 | 内容供给+2000万曝光 |
| C级 | 100人 | ¥0.5万 | ¥50万 | 口碑铺量+1000万曝光 |
| KOC | 300人 | ¥0.05万（产品成本） | ¥15万 | 素人口碑+500万曝光 |
| **合计** | **440人** | - | **¥249万** | **1.15亿+曝光** |

### 3.3 达人筛选标准

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| 粉丝质量 | 30% | 粉丝画像匹配度、活跃率、真实粉丝占比 |
| 内容质量 | 25% | 内容原创度、垂直度、互动率、爆文率 |
| 商业配合度 | 20% | 过往合作评价、内容交付准时率、修改配合度 |
| 品牌契合度 | 15% | 个人形象与品牌调性匹配、价值观一致性 |
| 性价比 | 10% | CPE/CPM成本、历史ROI数据 |

### 3.4 合作内容规范

| 内容类型 | 必含元素 | 禁止内容 |
|----------|----------|----------|
| 产品测评 | 真实使用场景、功能演示、个人感受 | 过度夸大、虚假数据 |
| 生活分享 | 自然植入、真实记录、情感共鸣 | 硬广口吻、过度P图 |
| 创作者故事 | 成长历程、平台对比、收益展示 | 贬低竞品、不实收益 |
| 活动参与 | 活动规则、参与方式、奖励展示 | 误导用户、虚假奖励 |

### 3.5 达人合作SOP

```
Week 1: 达人筛选与邀约
  -> 根据 campaign 需求筛选达人名单
  -> 发送合作邀约，确认合作意向
  -> 签署合作协议，支付预付款

Week 2: 内容策划与创作
  -> 提供Brief，明确内容方向与必含元素
  -> 达人提交内容大纲，审核通过
  -> 达人创作内容，提交初稿

Week 3: 内容审核与优化
  -> 内容审核（品牌合规+法务审核）
  -> 反馈修改意见，达人优化
  -> 终稿确认，排期发布

Week 4: 发布与复盘
  -> 按时发布，配合DOU+/薯条加热
  -> 数据监测，收集用户反馈
  -> 效果复盘，结算尾款
```

## 四、话题营销方案

### 4.1 核心话题标签体系

#### 话题1：#榔头一下#（品牌主话题）

| 维度 | 规划 |
|------|------|
| 话题定位 | 品牌核心话题，所有内容的统一标签 |
| 话题内涵 | "榔头一下"=打开榔头App，记录/发现真实生活 |
| 使用场景 | 所有品牌内容、用户UGC、KOL合作内容 |
| 传播目标 | 3个月话题阅读量1亿+，6个月5亿+ |
| 运营动作 | 每日发布带话题内容，每周话题互动活动 |

#### 话题2：#不装生活#（差异化话题）

| 维度 | 规划 |
|------|------|
| 话题定位 | 品牌差异化话题，对抗"精致焦虑" |
| 话题内涵 | 真实、自然、不伪装的生活方式 |

## 六、危机公关预案

### 6.1 危机分级体系

| 级别 | 定义 | 响应时间 | 决策层级 |
|------|------|----------|----------|
| **一级（红色）** | 重大负面事件（数据泄露、严重安全事故、高管丑闻） | 30分钟 | CEO+公关总监 |
| **二级（橙色）** | 较大负面事件（产品故障、用户投诉集中爆发、KOL翻车） | 2小时 | 公关总监+市场总监 |
| **三级（黄色）** | 一般负面事件（个别用户投诉、内容审核争议、 minor舆情） | 4小时 | 公关经理+运营经理 |
| **四级（蓝色）** | 潜在风险（竞品攻击、行业负面、用户误解） | 24小时 | 公关专员 |

### 6.2 危机响应流程

```
[监测发现] -> [初步评估] -> [分级定级] -> [启动预案] -> [响应处置] -> [持续监测] -> [复盘总结]
    |            |            |            |            |            |            |
  舆情系统     公关团队     危机小组     对应级别     官方声明     舆情跟踪     经验沉淀
  7x24监测     30min内      1h内决策     预案执行     渠道发布     每日报告     制度优化
```

### 6.3 常见危机场景与应对

#### 场景1：内容审核争议

| 环节 | 应对措施 |
|------|----------|
| 监测 | 舆情系统监测"榔头+审核/封号/误判"关键词 |
| 响应 | 4小时内联系用户，核实情况 |
| 处置 | 如确为误判，立即解封+补偿；如合规，耐心解释 |
| 声明 | 官方账号发布审核标准说明，透明化处理 |

#### 场景2：用户数据安全质疑

| 环节 | 应对措施 |
|------|----------|
| 监测 | 监测"榔头+数据/隐私/泄露"关键词 |
| 响应 | 30分钟内启动技术排查 |
| 处置 | 如确认问题，立即修复+通知用户；如谣言，发布技术说明 |
| 声明 | 发布数据安全白皮书，邀请第三方审计 |

#### 场景3：KOL合作翻车

| 环节 | 应对措施 |
|------|----------|
| 监测 | 监测合作KOL舆情，建立预警机制 |
| 响应 | 2小时内评估影响，决定是否终止合作 |
| 处置 | 立即下架相关合作内容，终止合作 |
| 声明 | 官方声明表明立场，强调品牌价值观 |

#### 场景4：竞品恶意攻击

| 环节 | 应对措施 |
|------|----------|
| 监测 | 监测竞品动态及负面信息来源 |
| 响应 | 24小时内收集证据，评估法律途径 |
| 处置 | 不主动回应攻击，以优质内容反击 |
| 声明 | 必要时发布声明，强调产品优势 |
