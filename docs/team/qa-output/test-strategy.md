# 榔头（Langtou）内容社区平台 - 测试策略与计划

## 1. 项目概述

榔头（Langtou）是一个对标小红书的图文/短视频内容社区平台，包含用户管理、内容发布、互动社交、消息通知等核心功能。本测试策略与计划旨在建立覆盖全链路的测试体系，保障平台质量与用户体验。

---

## 3. 测试策略

### 3.1 左移测试（Shift-Left Testing）

- **需求评审阶段**：测试人员参与需求评审，识别可测试性与风险点
- **设计阶段**：输出测试方案与用例设计，进行静态代码审查
- **开发阶段**：开发人员编写单元测试，测试人员准备自动化脚本
- **提测前**：执行冒烟测试，确保基本功能可用

### 3.2 自动化优先

| 层级 | 自动化策略 | 工具/框架 |
|------|-----------|-----------|
| 单元测试 | 强制要求，覆盖率目标 >= 80% | JUnit 5 + Mockito |
| 接口测试 | 核心接口 100% 覆盖 | REST Assured / JMeter |
| 性能测试 | 全量场景自动化 | Locust / JMeter |
| UI 测试 | P0 场景自动化 | Appium / Detox |

### 3.3 分层测试金字塔

```
        /\
       /  \   UI 自动化测试（10%）
      /----\
     /      \  接口自动化测试（30%）
    /--------\
   /          \ 单元测试（60%）
  /------------\
```

- **单元测试**：聚焦 Service/Mapper 层，快速反馈，执行时间 < 5 分钟
- **接口测试**：聚焦 Controller 层与跨服务调用，执行时间 < 15 分钟
- **UI 测试**：聚焦核心用户路径，执行时间 < 30 分钟
- **端到端测试**：关键业务流程全链路验证

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 测试环境不稳定 | 进度延迟 | 建立环境监控与快速恢复机制 |
| 自动化脚本维护成本高 | 效率降低 | 建立 Page Object / 数据驱动模式 |
| 性能瓶颈定位困难 | 调优周期长 | 提前部署 APM（SkyWalking/Pinpoint） |
| 需求频繁变更 | 用例失效 | 加强需求变更管控，核心用例优先覆盖 |
| 短视频场景测试复杂 | 覆盖不足 | 引入视频质量评估工具与真机测试 |
