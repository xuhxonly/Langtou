## 1. 版本背景与目标

### 1.1 版本背景

榔头(Langtou)内容社区平台当前处于 v6.7.0 版本，已具备用户体系、内容CRUD、基础社交互动、搜索推荐、广告系统等核心能力。然而，根据竞品分析和用户反馈，平台在以下方面存在明显短板：

- **推荐算法**: 仍采用基础协同过滤，缺乏个性化能力，用户留存率低于行业均值
- **内容审核**: 依赖纯人工审核，效率低、成本高，存在内容安全风险
- **电商能力**: 完全缺失，创作者变现渠道单一，平台商业化受限
- **合规能力**: 缺乏青少年模式、GDPR合规等基础能力

v7.0 版本作为平台战略升级的关键版本，旨在补齐上述核心短板，为后续差异化竞争奠定基础。

### 1.2 版本目标

| 目标类型 | 具体目标 | 衡量指标 |
|----------|----------|----------|
| **用户增长** | 提升用户留存和活跃度 | 7日留存率从25%提升至35%，人均使用时长从15分钟提升至25分钟 |
| **内容生态** | 提升内容质量和分发效率 | 内容审核时效从24小时缩短至2小时，优质内容曝光占比提升30% |
| **商业化** | 建立电商基础能力 | 上线商品橱窗功能，引入100+品牌/创作者入驻 |
| **合规安全** | 建立基础合规能力 | 上线青少年模式，通过等保三级初评 |

### 1.3 目标用户

| 用户群体 | 描述 | 核心诉求 |
|----------|------|----------|
| **内容消费者** | 18-35岁，一二线城市，追求生活品质 | 发现感兴趣的内容，获得消费决策参考 |
| **内容创作者** | 有一定专业能力的KOL/KOC | 获得流量和变现，建立个人品牌 |
| **品牌商家** | 中小品牌和新消费品牌 | 精准触达目标用户，实现种草转化 |
| **平台运营** | 榔头运营团队 | 高效管理内容和用户，数据驱动决策 |

### 3.1 推荐算法系统（深入设计）

#### 3.1.1 功能概述

建设榔头个性化推荐系统，实现从"千人一面"到"千人千面"的升级，提升内容分发效率和用户满意度。

#### 3.1.2 用户故事

| 编号 | 用户故事 | 验收标准 |
|------|----------|----------|
| RS-US-01 | 作为用户，我希望首页推荐的内容符合我的兴趣，以便快速找到感兴趣的内容 | 推荐内容与用户兴趣标签匹配度>70% |
| RS-US-02 | 作为用户，我希望推荐内容能实时反映我的最新行为，以便发现新兴趣 | 用户行为反馈后5分钟内影响推荐结果 |

#### 3.1.4 功能模块设计

##### 3.1.4.1 用户画像系统

**功能描述**: 构建多维度用户画像，为推荐算法提供用户特征输入。

**画像维度**:

| 维度类别 | 具体标签 | 数据来源 | 更新频率 |
|----------|----------|----------|----------|
| **基础属性** | 年龄/性别/地域/设备 | 注册信息/设备信息 | 实时 |
| **兴趣标签** | 内容分类偏好(一级/二级) | 点击/收藏/搜索行为 | 实时+日更新 |
| **行为特征** | 活跃时段/使用频次/消费深度 | 埋点数据 | 实时 |
| **社交特征** | 关注关系/互动偏好/粉丝属性 | 社交行为数据 | 小时级 |
| **消费特征** | 价格敏感度/品类偏好/购买频次 | 电商行为数据 | 实时 |
| **创作者属性** | 是否创作者/创作领域/粉丝数 | 创作者数据 | 日更新 |

**技术方案**:
- 存储: Redis(热数据) + HBase(冷数据) + ES(检索)
- 计算: Spark离线批处理 + Flink实时流处理
- 标签体系: 三级分类，共200+兴趣标签

##### 3.1.4.2 召回层

**功能描述**: 从海量内容中快速召回候选集，缩小排序范围。

**召回策略**:

| 召回通道 | 算法 | 召回数量 | 适用场景 |
|----------|------|----------|----------|
| 协同过滤 | ItemCF/UserCF | 500 | 用户有历史行为 |
| 向量召回 | 内容Embedding | 500 | 兴趣探索 |
| 热度召回 | 加权热度排序 | 200 | 新用户/冷启动 |
| 关注召回 | 关注作者新内容 | 200 | 社交关系 |
| 运营召回 | 人工配置 | 100 | 活动/价值观 |

**向量召回技术方案**:
- 内容向量化: 基于BERT+ResNet的多模态Embedding
- 向量检索: Milvus向量数据库，支持ANN近似最近邻搜索
- 向量维度: 512维
- 索引更新: 小时级增量更新

##### 3.1.4.3 排序层

**功能描述**: 对召回候选集进行精细排序，输出最终推荐列表。

**排序模型**:

| 模型 | 目标 | 特征 | 更新频率 |
|------|------|------|----------|
| 粗排模型 | 快速筛选 | 用户画像+内容统计特征 | 天级 |
| 精排模型 | 精准预测 | 200+特征，深度模型 | 小时级 |
| 重排模型 | 多样性/新鲜度 | 业务规则+多样性约束 | 实时 |

**精排模型设计**:
- 模型架构: DeepFM + DIN(深度兴趣网络)
- 预测目标: 点击率(CTR) + 互动率 + 停留时长(多目标优化)
- 特征工程:
  - 用户特征: 画像标签、历史行为统计、实时行为
  - 内容特征: 类别、标签、质量分、创作者等级、发布时间
  - 交叉特征: 用户-内容兴趣匹配度、用户-创作者关系
- 训练数据: 近30天曝光点击数据，日更新

**多目标优化**:
```
Score = w1 * pCTR + w2 * pLikeRate + w3 * pCommentRate + w4 * pStayTime
```
权重通过线上A/B测试动态调整。

##### 3.1.4.4 实时反馈机制

**功能描述**: 实时捕捉用户行为，动态调整推荐结果。

**实时数据流**:
```
用户行为(点击/点赞/收藏/滑动) 
  -> Kafka 
  -> Flink实时处理 
  -> 更新用户实时兴趣 
  -> 影响下次推荐
```

**实时特征**:

| 特征 | 计算逻辑 | 延迟要求 |
|------|----------|----------|
| 实时兴趣 | 近30分钟点击内容的标签聚合 | < 1分钟 |
| 会话上下文 | 当前会话已浏览内容的主题 | < 30秒 |
| 负反馈 | 用户"不感兴趣"标记的内容标签 | 实时 |
| 实时热度 | 近1小时内容互动量排序 | < 5分钟 |

##### 3.1.4.5 冷启动策略

**新用户冷启动**:

| 阶段 | 策略 | 内容来源 |
|------|------|----------|
| 0-1次访问 | 热门内容+多样性探索 | 全站热度Top+分类均衡 |
| 2-5次访问 | 兴趣试探+行为学习 | 基于点击反馈调整 |
| 5次+ | 个性化推荐 | 用户画像驱动 |

**新内容冷启动**:

| 阶段 | 策略 | 曝光机制 |
|------|------|----------|
| 发布0-30分钟 | 保底曝光池 | 向1000个潜在兴趣用户曝光 |
| 30分钟-2小时 | 效率测试 | 根据CTR决定是否放大 |
| 2小时+ | 正常推荐 | 进入常规推荐流程 |

### 3.2 AI内容审核系统（概要设计）

#### 3.2.1 功能概述
建设AI+人工协同的内容审核系统，提升审核效率和覆盖度，降低内容安全风险。

#### 3.2.2 审核流程

```
内容提交
  -> AI预审(秒级)
    -> 高置信度通过 -> 直接发布
    -> 高置信度拒绝 -> 直接拦截
    -> 中置信度 -> 人工复核队列
    -> 低置信度(新类型) -> 人工审核队列
  -> 人工审核(分钟级-小时级)
    -> 通过 -> 发布
    -> 拒绝 -> 拦截+通知用户
    -> 打回修改 -> 返回用户
```

#### 3.2.3 AI审核能力

| 审核类型 | 检测内容 | 技术方案 | 准确率目标 |
|----------|----------|----------|------------|
| 图像审核 | 色情/暴力/违禁品/广告二维码 | 基于YOLO+ResNet的目标检测 | > 95% |
| 文本审核 | 敏感词/垃圾广告/辱骂/涉政 | 基于BERT的文本分类 | > 92% |
| 视频审核 | 关键帧图像+ASR文本+音频 | 帧采样+图像审核+语音转文本 | > 90% |
| OCR审核 | 图片中的文字内容 | OCR+文本审核 | > 88% |

#### 3.2.4 人机协同平台

**审核后台功能**:
- 审核队列管理(按优先级/类型分配)
- 审核操作(通过/拒绝/打回/标记)
- 审核效率统计(人均审核量/准确率/时效)
- 申诉处理
- 审核规则配置

### 3.3 电商带货系统（概要设计）

#### 3.3.1 功能概述
建设基础电商能力，支持创作者通过内容带货变现，实现平台商业化升级。

#### 3.3.2 系统模块

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 商品管理 | 商品发布/编辑/上下架/类目管理 | P0 |
| 商品橱窗 | 创作者个人主页商品展示 | P0 |

#### 3.3.3 核心流程

**创作者带货流程**:
```
创作者入驻 -> 选品中心选品 -> 添加到自己的橱窗

**用户购买流程**:
```
浏览内容 -> 点击商品链接 -> 商品详情页 -> 下单支付

### 3.4 青少年模式（概要设计）

#### 3.4.1 功能概述
建设青少年模式，限制未成年用户的使用时长和内容访问，满足合规要求。

#### 3.4.2 核心功能

| 功能 | 说明 |
|------|------|
| 模式开启 | 实名认证年龄<18岁自动开启，或家长手动开启 |
