## 1. 背景与目标

### 1.1 背景

根据国家网信办《未成年人网络保护条例》及相关法规要求，面向未成年人的互联网产品必须建立青少年保护机制。同时，作为内容社区平台，榔头有责任保护未成年用户的身心健康，引导其合理使用互联网。

当前榔头平台在未成年人保护方面存在以下不足：

- **无年龄识别机制**: 无法区分成年用户与未成年用户
- **无内容分级**: 所有内容对全部用户可见，存在未成年人接触不适宜内容的风险
- **无使用时长限制**: 缺乏防沉迷机制
- **无家长管控**: 家长无法了解和管理孩子的使用行为

### 1.2 目标

| 目标类型 | 具体目标 | 衡量指标 |
|----------|----------|----------|
| **合规达标** | 满足《未成年人网络保护条例》要求 | 通过合规审查，无监管处罚 |
| **内容安全** | 确保未成年用户仅接触适龄内容 | 内容过滤准确率 > 98% |
| **防沉迷** | 限制未成年用户使用时长 | 使用时长合规率 > 95% |
| **家长满意** | 提供家长管控能力 | 家长控制面板使用率 > 30% |

### 1.3 目标用户

| 用户群体 | 描述 | 核心诉求 |
|----------|------|----------|
| **未成年用户** | <18岁的平台注册用户 | 安全使用平台，获取有益内容 |
| **家长/监护人** | 未成年用户的家长 | 了解和管理孩子的使用行为 |
| **平台运营** | 榔头运营团队 | 高效管理内容分级，确保合规 |
| **内容创作者** | 平台内容生产者 | 了解内容分级标准，正确标注内容 |

## 4. 详细功能设计

### 4.1 实名认证与年龄识别

#### 4.1.1 认证流程

```
用户首次使用平台
  -> 弹出实名认证引导（非强制，但青少年模式依赖此功能）
  -> 用户输入: 姓名 + 身份证号
  -> 后端调用公安实名认证接口验证
    -> 验证通过
      -> 解析出生日期，计算年龄
      -> 更新 User 实体: birthDate, isTeenager, teenModeEnabled
      -> <18岁: 自动开启青少年模式，弹出提示
      -> >=18岁: 正常使用
    -> 验证失败
      -> 提示"认证失败，请检查信息"
  -> 用户跳过认证
    -> 默认按成年用户处理
    -> 后续可在"设置-账号安全"中完成认证
```

#### 4.1.2 认证规则

| 规则 | 说明 |
|------|------|
| 认证时机 | 首次登录时引导，非强制但推荐 |
| 认证方式 | 姓名 + 身份证号，调用公安接口核验 |
| 年龄计算 | 基于身份证号解析出生日期，与当前日期计算周岁 |
| 未成年人定义 | 周岁 < 18 岁 |
| 认证修改 | 每个身份证号仅可认证一次，不可更改 |
| 隐私保护 | 身份证号加密存储，符合个人信息保护法要求 |

### 4.2 内容分级体系

#### 4.2.1 分级标准

| 等级 | 标识 | 适用年龄 | 内容范围 | 示例 |
|------|------|----------|----------|------|
| **0 - 全年龄** | 🟢 全年龄 | 所有用户 | 教育、科普、正能量、生活技巧 | 数学学习、科学实验、正能量故事 |

| **12+** | 🟠 12岁以上 | >= 12岁 | 适合青少年的内容，包含一定深度 | 学科辅导、时事讨论、健身教程 |

### 4.3 使用时长管理

#### 4.3.1 时长限制规则

| 规则 | 默认值 | 家长可配置 |
|------|--------|-----------|
| 每日使用时长上限 | 40分钟 | 15 / 30 / 40 / 60 分钟 |
| 夜间禁用时段 | 22:00 - 06:00 | 不可配置（法规要求） |
| 超时提醒时间 | 剩余5分钟 | 不可配置 |
| 休息间隔提醒 | 连续使用20分钟 | 不可配置 |

#### 4.3.2 时长管理流程

```
用户打开App
  -> 检查当前时间
    -> 22:00-06:00: 显示夜间禁用页面，禁止进入
    -> 其他时间: 正常进入
  -> 检查今日已使用时长
    -> 已达上限: 显示"今日使用时长已到"页面，禁止进入
    -> 未达上限: 正常使用
  -> 使用过程中
    -> 每分钟更新使用时长
    -> 剩余5分钟: 弹出提醒"还有5分钟，请注意休息"
    -> 剩余1分钟: 弹出提醒"还有1分钟，请做好准备"
    -> 达到上限: 弹出"今日使用时长已到，明天再来吧"，强制退出
    -> 连续使用20分钟: 弹出"已使用20分钟，建议休息一下眼睛"
```

## 5. 技术方案

### 5.1 数据模型变更

#### 5.1.1 User 实体新增字段

```sql
ALTER TABLE users ADD COLUMN birth_date DATE COMMENT '出生日期（实名认证后填写）';
ALTER TABLE users ADD COLUMN is_teenager TINYINT(1) DEFAULT 0 COMMENT '是否未成年（<18岁）';
ALTER TABLE users ADD COLUMN teen_mode_enabled TINYINT(1) DEFAULT 0 COMMENT '青少年模式是否开启';
ALTER TABLE users ADD COLUMN id_card_encrypted VARCHAR(255) COMMENT '加密存储的身份证号';
ALTER TABLE users ADD COLUMN real_name VARCHAR(50) COMMENT '实名认证姓名';
ALTER TABLE users ADD COLUMN identity_verified TINYINT(1) DEFAULT 0 COMMENT '实名认证状态';
ALTER TABLE users ADD COLUMN parent_password_hash VARCHAR(255) COMMENT '家长密码哈希';
ALTER TABLE users ADD COLUMN daily_time_limit INT DEFAULT 40 COMMENT '每日使用时长限制（分钟）';
ALTER TABLE users ADD COLUMN teen_mode_created_at DATETIME COMMENT '青少年模式开启时间';
```

#### 5.1.2 Content 实体新增字段

```sql
ALTER TABLE contents ADD COLUMN age_rating TINYINT DEFAULT 0 COMMENT '年龄分级：0-全年龄/1-6+/2-12+/3-18+';
ALTER TABLE contents ADD COLUMN age_rating_source VARCHAR(20) DEFAULT 'auto' COMMENT '分级来源：auto/creator/manual';
ALTER TABLE contents ADD COLUMN age_rating_verified TINYINT(1) DEFAULT 0 COMMENT '分级是否经人工审核';
```

#### 5.1.3 新增数据表

**用户每日使用时长记录表**:

```sql
CREATE TABLE teen_daily_usage (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
```

### 5.2 后端中间件设计

#### 5.2.1 青少年模式中间件

```
请求进入
  -> TeenModeMiddleware
    -> 检查用户是否登录
      -> 未登录: 放行（游客模式）
    -> 检查用户 teenModeEnabled
      -> 未开启: 放行
      -> 已开启:
        -> 检查当前时间
          -> 22:00-06:00: 返回夜间禁用响应
        -> 检查今日使用时长
          -> 已超限: 返回超时响应
        -> 检查请求类型
          -> 内容浏览请求: 注入 ageRating 过滤条件
          -> 禁止功能请求: 返回功能限制响应
          -> 其他请求: 放行
    -> 记录使用时长（异步）
```

#### 5.2.2 内容过滤实现

```python
# 伪代码示例
class TeenModeMiddleware:
    def process_request(self, request):
        user = request.user
        if not user or not user.teen_mode_enabled:
            return None  # 非青少年模式，放行

        # 夜间禁用检查
        now = datetime.now()
        if self.is_night_time(now):
            return TeenModeResponse.NIGHT_RESTRICTION

        # 使用时长检查
        usage = self.get_daily_usage(user.id)
        limit = user.daily_time_limit * 60  # 转换为秒
        if usage.total_seconds >= limit:
            return TeenModeResponse.TIME_LIMIT

        # 功能限制检查
        if self.is_restricted_action(request):
            return TeenModeResponse.FEATURE_RESTRICTED

        # 内容过滤注入
        if self.is_content_request(request):
            request.teen_mode_filter = {
                'max_age_rating': self.get_max_rating(user)
            }

        return None  # 放行

    def get_max_rating(self, user):
        age = self.calculate_age(user.birth_date)
        if age < 6:
            return 0   # 仅全年龄
        elif age < 12:
            return 1   # 6+
        elif age < 18:
            return 2   # 12+
        return 3       # 18+（理论上不会到这里）
```

### 5.3 前端实现方案

#### 5.3.1 使用时长统计

```
前端使用时长统计逻辑:
  1. App进入前台时，记录 session_start_time
  2. 每分钟心跳上报: POST /api/v1/teen/heartbeat
     - 携带 session_duration（本次前台时长）
  3. App进入后台时，上报本次前台使用时长

  5. 前端定时（每30秒）轮询剩余时长: GET /api/v1/teen/usage
```

## 9. 风险评估

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 实名认证接口不稳定 | 用户无法完成认证 | 中 | 准备多个认证服务商，降级方案支持手动上传证件照 |
| 内容分级标注不准 | 未成年人接触不适宜内容 | 中 | AI标注+人工审核双重保障，建立用户举报机制 |
| 未成年人使用家长身份注册 | 绕过青少年模式 | 高 | 结合人脸识别辅助验证，持续优化风控策略 |
| 时长统计被绕过 | 防沉迷失效 | 低 | 多端统计+服务端校验，异常使用行为检测 |
| 家长密码被破解 | 未成年人自行关闭模式 | 低 | 连续错误锁定，密码复杂度要求，找回需实名验证 |
