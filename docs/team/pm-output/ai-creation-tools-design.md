## 1. 概述

### 1.1 背景

榔头作为内容社区平台，降低创作门槛、提升创作效率是提升内容供给质量和数量的关键。AI创作工具通过智能化的文案生成、标签推荐和封面选择能力，帮助创作者快速产出高质量内容，缩短从"想法"到"发布"的路径。

### 1.2 目标

- **降低创作门槛**：从图片到完整笔记，AI辅助完成80%的文案工作
- **提升内容质量**：通过AI优化标题、标签和封面，提升内容曝光和互动率
- **提升创作效率**：将平均创作时间从15分钟缩短至5分钟
- **增强创作体验**：提供多样化、个性化的AI辅助选项

### 1.3 功能范围

| 功能模块 | 优先级 | 说明 |
|---------|--------|------|
| AI标题生成 | P0 | 根据图片自动生成3-5个候选标题 |
| AI标签推荐 | P0 | 根据图片内容推荐5-10个话题标签 |
| AI文案草稿 | P1 | 根据标题生成正文草稿 |
| 智能封面选择 | P1 | 从多张图片中推荐最佳封面 |
| 文案风格选择 | P1 | 种草/测评/教程/Vlog四种风格 |

### 1.4 用户权限

| 用户等级 | AI生成次数/天 | 说明 |
|---------|-------------|------|
| 普通用户 | 10次 | 基础创作辅助 |
| 认证创作者 | 50次 | 认证后提升配额 |
| 付费会员 | 不限 | VIP无限使用 |

## 2. AI文案助手

### 2.1 AI标题生成

**功能描述**：用户上传图片后，AI根据图片内容自动生成3-5个候选标题，用户可选择其中一个或在此基础上修改。

**输入**：
- 上传的图片（1张或多张）
- 可选：用户输入的关键词/主题偏好

**输出**：
- 3-5个候选标题
- 每个标题附带推荐理由

**标题生成规则**：

| 规则 | 说明 |
|-----|------|
| 字数限制 | 10-30个字符 |
| 风格多样性 | 同一批生成结果覆盖不同风格（种草/信息/情感） |
| 热点感知 | 结合当前平台热门话题和趋势 |
| SEO友好 | 包含高频搜索关键词 |
| 去重 | 避免与已有笔记标题高度重复 |

**标题示例**：

```
输入图片: 一张咖啡店拉花咖啡的照片

输出候选标题:
1. "藏在胡同里的宝藏咖啡馆，拿铁拉花也太美了！" (种草风格)
2. "北京胡同咖啡店探店 | 拿铁拉花艺术全记录" (测评风格)
3. "一杯咖啡的仪式感：手把手教你拍出ins风咖啡照" (教程风格)
4. "周末去哪儿？这家咖啡店让我待了一整个下午" (Vlog风格)
5. "咖啡控必看！胡同深处的意外惊喜" (情感风格)
```

**推荐理由展示**：
```
标题1: "藏在胡同里的宝藏咖啡馆，拿铁拉花也太美了！"
推荐理由: 使用了"宝藏""也太"等高频种草词汇，容易引发好奇和互动
```

### 2.2 AI标签推荐

**功能描述**：根据图片内容和标题，AI推荐5-10个话题标签，帮助笔记获得更精准的流量分发。

**输入**：
- 上传的图片
- 已选标题（如有）
- 用户手动输入的标签（作为补充参考）

**输出**：
- 5-10个推荐标签
- 按相关度排序
- 标注标签热度（热门/普通/长尾）

**标签分类**：

| 分类 | 说明 | 示例 |
|-----|------|------|
| 核心标签 | 与图片内容直接相关 | #咖啡 #拿铁 #拉花 |
| 场景标签 | 描述图片场景 | #探店 #周末去哪儿 #下午茶 |
| 风格标签 | 描述内容风格 | #ins风 #日系 #复古 |
| 情绪标签 | 描述情感氛围 | #治愈 #小确幸 #慢生活 |
| 热门标签 | 平台当前热门话题 | #夏日饮品 #城市漫游 |

**标签推荐示例**：

```
输入图片: 咖啡店拉花咖啡照片 + 标题"藏在胡同里的宝藏咖啡馆"

推荐标签:
1. #咖啡探店        [核心] [热门 - 日均曝光10w+]
2. #拿铁拉花        [核心] [普通]
3. #宝藏店铺        [场景] [热门 - 日均曝光8w+]
4. #胡同咖啡馆      [场景] [长尾]
5. #下午茶时光      [场景] [热门 - 日均曝光6w+]
6. #ins风咖啡       [风格] [普通]
7. #治愈系          [情绪] [热门 - 日均曝光5w+]
8. #北京探店        [场景] [热门 - 日均曝光7w+]
9. #咖啡控         [核心] [普通]
10. #慢生活         [情绪] [普通]
```

### 2.3 AI文案草稿生成

**功能描述**：根据用户选择的标题和上传的图片，AI生成完整的笔记正文草稿。

**输入**：
- 已选标题
- 上传的图片
- 文案风格（种草/测评/教程/Vlog）
- 可选：用户补充的关键信息（地址、价格、体验等）

**输出**：
- 完整的文案草稿（300-800字）
- 包含段落结构和emoji建议

**文案风格定义**：

#### 种草风格
```
特点: 感性表达，强调个人体验和推荐
语气: 热情、亲切、有感染力
结构: 引子(吸引注意) → 体验描述(细节) → 推荐(行动号召)
示例:
"姐妹们！！这家店我真的会反复去！
藏在胡同里的宝藏咖啡馆，推开门就是满满的ins风..."
```

#### 测评风格
```
特点: 客观分析，有理有据，信息量大
语气: 专业、理性、有条理
结构: 概述 → 详细测评(多维度) → 总结评分
示例:
"今天给大家测评一家胡同深处的独立咖啡馆。
环境: 4.5/5 - 胡同老房改造，保留了原始建筑特色..."
```

#### 教程风格
```
特点: 步骤清晰，实用性强，可操作
语气: 耐心、细致、鼓励性
结构: 前言 → 准备/背景 → 分步教程 → 注意事项
示例:
"想拍出好看的咖啡照片？跟着我一步步来！
Step 1: 选择靠窗的位置，利用自然光..."
```

#### Vlog风格
```
特点: 叙事性强，有故事线，代入感强
语气: 自然、口语化、像和朋友聊天
结构: 开场(设定场景) → 过程(叙事) → 结尾(感悟)
示例:
"周末睡到自然醒，决定出门走走。
偶然拐进一条胡同，发现了一家超有感觉的咖啡馆..."
```

### 2.4 文案风格选择交互

```
用户选择文案风格
  │
  ├─ [种草] 图标: 火焰/推荐
  │   描述: 热情推荐，适合分享好物和体验
  │
  ├─ [测评] 图标: 星星/评分
  │   描述: 客观分析，适合详细评测和对比
  │
  ├─ [教程] 图标: 书本/步骤
  │   描述: 步骤清晰，适合分享技巧和攻略
  │
  └─ [Vlog] 图标: 摄像机/记录
      描述: 叙事记录，适合记录日常和旅行
```

## 3. 智能封面选择

### 3.1 功能描述

用户上传多张图片时，AI自动分析每张图片的构图质量，推荐最适合作为笔记封面的图片，并提供一键裁剪为平台推荐比例（3:4）的功能。

### 3.2 封面构图评分

**评分维度**：

| 维度 | 权重 | 评分标准 |
|-----|------|---------|
| 主体突出 | 35% | 主体是否清晰、居中或符合构图法则 |
| 色彩丰富度 | 25% | 色彩饱和度、对比度、色彩搭配 |
| 清晰度 | 20% | 图像锐度、噪点水平、分辨率 |
| 构图美感 | 20% | 三分法、黄金比例、对称性等构图规则 |

**评分输出**：

```json
{
  "images": [
    {
      "image_index": 0,
      "image_url": "https://cdn.langtou.com/temp/img_001.jpg",
      "overall_score": 92,
      "scores": {
        "subject_clarity": 95,
        "color_richness": 88,
        "sharpness": 93,
        "composition": 90
      },
      "is_recommended": true,
      "recommend_reason": "主体清晰突出，色彩搭配和谐，构图符合三分法"
    },
    {
      "image_index": 1,
      "image_url": "https://cdn.langtou.com/temp/img_002.jpg",
      "overall_score": 78,
      "scores": {
        "subject_clarity": 75,
        "color_richness": 82,
        "sharpness": 70,
        "composition": 85
      },
      "is_recommended": false,
      "recommend_reason": "清晰度略低，建议使用第一张作为封面"
    }
  ]
}
```

### 3.3 一键裁剪

**裁剪规则**：

| 参数 | 说明 |
|-----|------|
| 目标比例 | 3:4 (平台推荐封面比例) |
| 裁剪策略 | 优先保留主体区域，智能填充/裁剪边缘 |
| 最大分辨率 | 1080 x 1440 px |
| 输出格式 | JPEG (质量85%) |

**裁剪流程**：

```
原始图片
  │
  ├─ AI识别图片主体区域 (目标检测)
  │
  ├─ 计算裁剪框
  │    ├─ 主体居中
  │    ├─ 保持3:4比例
  │    └─ 最大化利用图片面积
  │
  ├─ 用户可手动调整裁剪框
  │    ├─ 拖动裁剪框位置
  │    └─ 缩放裁剪框大小
  │
  └─ 确认裁剪 → 生成封面图
```

### 4.2 大语言模型接入

**模型选型**：

| 模型 | 提供商 | 用途 | 备选 |
|-----|--------|------|------|
| 通义千问-Max | 阿里云 | 标题生成、文案草稿 | 文心一言4.0 |
| 通义千问-VL | 阿里云 | 图片理解、标签推荐 | 文心一言-VL |
| 通义千问-Plus | 阿里云 | 低成本场景（标签推荐） | 文心一言3.5 |

**接入方式**：

```
AI创作服务 → HTTP API调用 → LLM Provider

调用参数:
- model: 模型标识
- messages: Prompt + 上下文
- temperature: 创造性参数 (标题: 0.8, 文案: 0.7, 标签: 0.5)
- max_tokens: 最大输出长度
- top_p: 核采样参数
```

**模型调用配置**：

| 场景 | 模型 | Temperature | Max Tokens | 超时时间 |
|-----|------|------------|-----------|---------|
| 标题生成 | qwen-max | 0.8 | 500 | 10s |
| 标签推荐 | qwen-plus | 0.5 | 300 | 8s |
| 文案草稿 | qwen-max | 0.7 | 2000 | 30s |
| 图片理解 | qwen-vl-max | 0.3 | 500 | 15s |
| 封面评分 | 自研视觉模型 | - | - | 5s |

### 4.3 多模态图片理解

**图片理解流程**：

```
用户上传图片
  │
  ├─ 1. 图片预处理
  │    ├─ 格式转换 (HEIC/WEBP → JPEG)
  │    ├─ 尺寸压缩 (最大2048px)
  │    ├─ 质量压缩 (JPEG质量80%)
  │    └─ 生成缩略图 (用于模型输入)
  │
  ├─ 2. 图片内容分析 (多模态模型)
  │    ├─ 主体识别: 咖啡、美食、穿搭、旅行等
  │    ├─ 场景识别: 室内、室外、餐厅、街道等
  │    ├─ 风格识别: 日系、复古、ins风等
  │    ├─ 色彩分析: 主色调、色彩搭配
  │    └─ 情感分析: 温暖、清新、活力等
  │
  ├─ 3. 内容结构化输出
  │    {
  │      "subject": "咖啡拉花",
  │      "scene": "咖啡馆",
  │      "style": "ins风",
  │      "colors": ["棕色", "白色", "奶咖色"],
  │      "mood": "温暖治愈",
  │      "objects": ["咖啡杯", "拉花", "木质桌面"],
  │      "description": "一杯精致的拿铁拉花咖啡，放在木质桌面上，背景是温馨的咖啡馆环境"
  │    }
  │
  └─ 4. 基于分析结果生成标题/标签/文案
```

### 4.5 生成结果缓存

**缓存策略**：

| 缓存类型 | 缓存Key | 过期时间 | 说明 |
|---------|---------|---------|------|
| 图片分析结果 | `ai:analysis:{image_hash}` | 24小时 | 相同图片不重复分析 |
| 标题生成结果 | `ai:title:{image_hash}:{keywords_hash}` | 6小时 | 相同图片+关键词复用 |
| 标签推荐结果 | `ai:tags:{image_hash}:{title_hash}` | 6小时 | 相同图片+标题复用 |
| 文案草稿结果 | `ai:draft:{title_hash}:{style}` | 1小时 | 相同标题+风格复用 |
| 封面评分结果 | `ai:cover:{image_hash}` | 24小时 | 相同图片不重复评分 |

**缓存实现**：

```
Redis缓存方案:

Key格式: ai:{type}:{content_hash}
Value: JSON序列化的生成结果
TTL: 根据缓存类型设置

图片Hash计算:
- 使用感知哈希(pHash)算法
- 对图片缩放后计算，忽略尺寸差异
- Hash长度: 64位

缓存命中逻辑:
1. 计算输入内容的Hash
2. 查询Redis缓存
3. 命中 → 直接返回缓存结果
4. 未命中 → 调用LLM API → 结果写入缓存 → 返回
```

## 8. 风控与安全

### 8.1 内容安全

| 风险点 | 防控措施 |
|-------|---------|
| AI生成违规内容 | Prompt中注入安全约束 + 生成结果过审核引擎 |
| AI生成虚假信息 | Prompt中强调"基于图片内容"，禁止编造 |
| 敏感话题标签 | 标签推荐结果过敏感词过滤 |
| 图片版权问题 | 提示用户确保拥有图片使用权 |

### 8.2 调用频率控制

| 维度 | 限制 | 说明 |
|-----|------|------|
| 单用户每分钟 | 最多3次AI调用 | 防止恶意刷接口 |
| 单用户每天 | 10/50/无限次 | 按用户等级分配 |
| 全局每秒 | 最多100次 | 服务端限流 |
| 单次生成超时 | 15秒 | 超时自动取消 |

### 8.3 数据隐私

- 用户上传的图片仅用于本次AI生成，不持久化存储
- 图片分析结果缓存24小时后自动清除
- AI调用日志仅记录元数据，不记录原始图片和生成内容
- 用户可随时删除已生成的AI内容
