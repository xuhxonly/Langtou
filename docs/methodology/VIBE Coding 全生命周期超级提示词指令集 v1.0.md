# VIBE Coding 全生命周期超级提示词指令集 v1.0

## 使用说明

### 核心理念

本指令集融合了《Vibe Coding 完整提示词库 v3.0》与《VIBE Coding 模式系统提示词大全》的全部精华，构建了一套覆盖软件产品从零到一、从一到N全生命周期的完整提示词体系。每一阶段均包含状态感知、决策逻辑与行动指令三大要素，使得AI编程助手能够根据当前代码库状态自动识别所处阶段，并调用相应的提示词推进项目进展。

### 使用方法

1. **状态感知**：AI首先对当前代码库进行全面扫描，识别已有哪些文件、模块、文档和配置

2. **阶段匹配**：根据感知结果，从本指令集中匹配当前所处的开发阶段

3. **任务执行**：调用对应的提示词指令，生成下一步需要完成的工作内容

4. **迭代循环**：每完成一个任务，重新评估状态，进入下一轮迭代，直至达到商品化标准

### 状态机总览

text

项目启动 → 竞品分析 → 系统初始化 → 团队构建 → 需求分析 → 架构设计 → 项目立项 
    → 迭代开发 → 诊断调试 → 测试验证 → 美术音效 → 运营数据 → 平台适配 
    → 部署发布 → 商品化评估 → 平台化 → 持续迭代 → 知识沉淀 → 风险管理 → 创新突破

### 指令 1.1：项目愿景定义

**触发条件**：代码库为空，用户提出了初步项目想法但未形成文档。

**执行指令**：

text

【角色】请以「项目战略顾问」身份执行以下任务：
【感知】检查当前目录下是否存在任何项目描述文档（如README.md、PROJECT_IDEA.md），若有，读取其内容作为项目构想基础；若无，向用户收集以下信息：

- 项目核心想法（1-3句话描述）
- 目标用户群体
- 要解决的核心问题
  【决策】基于感知到的信息，判断项目构想清晰度：
- 若信息充分（包含项目想法、目标用户、核心问题），直接进入「行动」步骤
- 若信息不足，先向用户提出10个关键问题以厘清想法
  【行动】生成以下交付物：
1. 《项目愿景说明书》，包含：
   - 项目名称建议（3-5个备选，含命名理由）
   - 项目类型定位（Web应用/移动应用/游戏/工具/平台）
   - 核心目标用户群体画像（年龄、场景、痛点）
   - 产品核心价值主张（解决什么核心问题）
   - 预期上线平台（Web/小程序/iOS/Android/桌面端）
   - 成功标准（3-5个可量化KPI，含目标值）
2. 在项目根目录创建 docs/vision/ 目录
3. 生成 PROJECT_VISION.md 文件
4. 创建 PROJECT_IDEA.md 作为初始想法记录
   【输出格式】先输出项目构想清晰度评估，再输出完整愿景说明书。
   【下一步】执行指令 1.2（项目可行性评估）

### 指令 1.2：项目可行性评估

**触发条件**：已存在 PROJECT_VISION.md 文件，但不存在 FEASIBILITY_REPORT.md 文件。

**执行指令**：

text

【角色】请以「项目战略顾问 + 技术专家」身份执行以下任务：
【感知】读取 PROJECT_VISION.md 获取项目构想；检查目录结构，评估已有技术储备。
【决策】从以下五个维度进行可行性评分（各维度1-10分，总分50分）：

- 若总分 ≥ 35分 → Go决策，进入「行动」
- 若总分 25-34分 → 有条件Go，列出需要补充的条件
- 若总分 < 25分 → No-Go决策，提出替代方案
  【行动】生成《项目可行性评估报告》，包含：
1. 技术可行性（现有技术栈能否实现，技术难点清单）
2. 市场可行性（目标市场大小、增长趋势估算）
3. 竞争可行性（红海/蓝海判断、进入壁垒分析）
4. 资源可行性（人力、时间、成本初步预估）
5. 风险可行性（主要风险及应对策略）
6. 综合可行性评分与Go/No-Go建议
7. 若为有条件Go，列出前置条件清单
   在 docs/analysis/ 目录下生成 FEASIBILITY_REPORT.md 文件。
   【输出格式】先输出可行性评分矩阵，再输出详细报告。
   【下一步】若结果为Go或条件Go，执行指令 2.1（竞品全面对标分析）；若结果为No-Go，结束流程并向用户说明理由。

### 指令 1.3：创新点挖掘

**触发条件**：已存在 PROJECT_VISION.md 且用户要求进行创新分析，或竞品分析完成后需要差异化定位。

**执行指令**：

text

【角色】请以「创新战略专家」身份执行以下任务：
【感知】读取 PROJECT_VISION.md 和 FEASIBILITY_REPORT.md；搜索该领域的常见解决方案模式。
【决策】基于行业分析和用户需求分析，识别未被满足的需求空间，筛选出最具潜力的5个创新方向。
【行动】生成《创新机会分析报告》：

1. 行业现有解决方案的痛点和不足（至少5个）
2. 未被满足的用户需求清单（至少5个）
3. 跨行业可借鉴的创新模式（至少3个行业）
4. 技术新可能分析（AI/WebGL/WebAssembly/区块链等）
5. 商业模式创新空间
6. 5个创新方向及其可行性评估（技术可行性、市场可行性、成本评估）
7. 每个创新方向的实施复杂度评级（低/中/高）
8. 优先推荐的2个创新方向及理由
   在 docs/analysis/ 目录下生成 INNOVATION_ANALYSIS.md 文件。
   【输出格式】创新方向以表格形式呈现，包含：方向名称、描述、可行性评分、实施建议。

## 第二篇章：市场洞察与竞争分析

### 指令 2.1：竞品全面对标分析

**触发条件**：已存在项目愿景文档，但不存在竞品分析报告；或距离上次竞品分析超过3个月。

**执行指令**：

text

【角色】请以「市场分析师 + 产品经理」身份执行以下任务：
【感知】读取 PROJECT_VISION.md 获取产品定位；搜索当前市场Top 10相关产品信息。
【决策】筛选出5个核心竞品进行深度分析，构建竞品对比矩阵。
【行动】执行竞品全面对标分析：

1. 搜索并列出该领域Top 10产品或服务（需验证信息的时效性）
2. 筛选5个核心竞品进行深度分析
3. 对每个竞品从以下维度打分（1-10分）：
   - 功能完整性
   - 用户体验
   - 视觉设计
   - 性能表现
   - 价格策略
   - 用户口碑（来自实际用户评论分析）
   - 创新程度
4. 制作竞品对比矩阵（雷达图/表格）
5. 分析用户评价情感倾向（正面/负面关键词提取）
6. 识别机会点和差距
7. 提出差异化竞争策略
   在 docs/analysis/ 目录下生成 COMPETITOR_ANALYSIS.md 和 DIFFERENTIATION_STRATEGY.md 文件。
   【输出格式】包含完整的竞品评分表、对比矩阵和差异化策略建议。

---

### 指令 2.2：用户需求深度调研

**触发条件**：已存在竞品分析报告，但不存在用户画像和用户旅程文档。

**执行指令**：

text

【角色】请以「用户体验研究员」身份执行以下任务：
【感知】读取 PROJECT_VISION.md 和 COMPETITOR_ANALYSIS.md，获取产品定位和竞品用户群体信息。
【决策】基于产品定位和目标市场，定义3-5个典型用户画像，识别其核心需求。
【行动】生成《用户需求调研报告》：

1. 定义3-5个典型用户画像（包含：姓名、年龄、职业、场景、痛点、动机、使用频率）
2. 绘制用户旅程地图（从认知到使用的完整路径，含5-7个关键触点）
3. 识别每个触点的痛点和机会点
4. 使用JTBD框架列出用户核心需求（至少10条，按重要性排序）
5. 区分"表面需求"和"深层需求"
6. 创建用户共情地图（说什么、想什么、做什么、感觉什么）
7. 提出基于用户洞察的产品建议（至少5条）
   在 docs/analysis/ 目录下生成 USER_RESEARCH.md 文件。
   【输出格式】用户画像需具象化（含虚拟姓名和场景描述），用户旅程需包含情绪曲线。

### 指令 2.3：市场规模与趋势分析

**触发条件**：已存在用户调研报告，但不存在市场分析报告。

**执行指令**：

text

### 指令 2.4：SWOT深度分析

**触发条件**：已存在竞品分析报告，但不存在SWOT分析文档。

**执行指令**：

text

【角色】请以「战略管理顾问」身份执行以下任务：
【感知】读取 PROJECT_VISION.md、COMPETITOR_ANALYSIS.md 和 MARKET_ANALYSIS.md。
【决策】基于内外部因素分析，制定4大战略方向。
【行动】生成《SWOT分析报告》：

1. 优势（Strengths）：我们做得比竞品好的地方（至少5条，每条含具体数据或论据）
2. 劣势（Weaknesses）：我们不如竞品的地方（至少5条）
3. 机会（Opportunities）：市场环境中可利用的机会（至少5条）
4. 威胁（Threats）：市场环境中需要应对的挑战（至少5条）
5. 与Top 3竞品的SWOT对比矩阵
6. 基于SWOT的4大战略方向：
   - SO战略（利用优势抓住机会）
   - WO战略（克服劣势利用机会）
   - ST战略（利用优势应对威胁）
   - WT战略（克服劣势应对威胁）
7. 各战略的优先级排序和资源需求评估
   在 docs/analysis/ 目录下生成 SWOT_ANALYSIS.md 文件。
   【输出格式】SWOT矩阵表格化，战略方向需包含具体行动建议和时间框架。

1. **状态感知**：AI首先扫描当前代码库和文档目录，识别已完成的阶段和缺失的产出物

2. **阶段匹配**：根据状态感知结果，从上述映射表中找到匹配的阶段

3. **指令选择**：从该阶段中选择最优先需要执行的指令

4. **执行交付**：执行指令并生成对应的交付物

5. **迭代循环**：交付完成后，重新评估状态，进入下一阶段
