# 🔒 安全政策

## 报告安全漏洞

我们认真对待安全问题。如果你发现了安全漏洞，请通过以下方式报告：

**邮箱**：security@langtou.com

**请包含以下信息**：
- 漏洞类型
- 受影响的版本
- 漏洞描述
- 复现步骤
- 潜在影响
- 修复建议（可选）

## 安全处理流程

1. **确认**：我们会在 48 小时内确认你的报告
2. **分析**：我们会在 1 周内分析漏洞的影响
3. **修复**：我们会尽快修复漏洞并发布补丁
4. **披露**：修复完成后，我们会在 GitHub Security Advisories 中披露漏洞信息

## 安全最佳实践

### 依赖安全
- 我们使用 **Dependabot** 自动检查依赖更新
- 定期运行安全扫描
- 及时应用安全补丁

### 代码安全
- 所有代码都会经过安全审查
- 使用静态代码分析工具
- 遵循 OWASP Top 10 最佳实践

### 数据安全
- 敏感数据加密存储
- 使用 HTTPS 传输数据
- 遵循数据最小化原则

## 负责任的披露

我们遵循负责任的披露原则：
- 给予足够的时间修复漏洞
- 不在修复前公开漏洞细节
- 感谢安全研究人员的贡献
