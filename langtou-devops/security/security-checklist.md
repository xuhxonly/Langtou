## 1. JWT 安全配置检查

| 检查项 | 要求 | 状态 | 备注 |
|--------|------|------|------|
| JWT Secret 强度 | 至少256位随机密钥 | [ ] | 使用 `openssl rand -base64 32` 生成 |
| JWT 签名算法 | 使用 RS256 或 ES256，禁止 HS256 | [ ] | 防止算法混淆攻击 |
| Token 有效期 | Access Token <= 2小时，Refresh Token <= 7天 | [ ] | |
| Token 黑名单 | 支持Token主动失效/注销 | [ ] | 通过Redis实现 |
| Token 存储 | HttpOnly Cookie 或 Authorization Header | [ ] | 禁止URL参数传递 |
| Token 刷新机制 | Refresh Token 轮转，旧Token立即失效 | [ ] | 防止Token重放 |
| JWT Payload | 不包含敏感信息(密码、身份证号等) | [ ] | |
| 多设备登录控制 | 支持踢出其他设备 | [ ] | |

| 检查项 | 要求 | 状态 | 备注 |
