# 🌱 贡献指南

感谢你对 Langtou 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 1. 报告 Bug
使用 GitHub Issues 的 Bug 模板报告问题。

### 2. 提交功能请求
使用 GitHub Issues 的功能请求模板提交建议。

### 3. 提交代码

#### 开发流程

```bash
# 1. Fork 本仓库
# 2. 克隆你的 Fork
git clone https://github.com/你的用户名/Langtou.git
cd Langtou

# 3. 创建功能分支
git checkout -b feature/你的功能名

# 4. 提交更改
git commit -m 'feat: 添加新功能'

# 5. 推送到你的 Fork
git push origin feature/你的功能名

# 6. 提交 Pull Request
```

#### 代码规范

- **Java**: 遵循 [阿里巴巴Java开发手册](https://github.com/alibaba/p3c)
- **JavaScript/TypeScript**: 遵循 ESLint 配置
- **Python**: 遵循 PEP 8 规范
- **通用**: 添加必要的注释，保持代码整洁

#### 提交规范

使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
feat: 新功能
fix: Bug 修复
docs: 文档更新
style: 代码格式
refactor: 代码重构
test: 添加测试
chore: 构建/工具
```

### 4. 代码审查

- 所有 PR 都会经过代码审查
- 审查重点：代码质量、安全性、性能、可维护性
- 审查时间：通常在 3 个工作日内

## 贡献者行为准则

### 我们的标准

- 尊重不同的意见和观点
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示善意

### 不可接受的行为

- 使用性化语言或图像
- 恶意攻击、人身攻击
- 发布私人信息
- 其他不专业的行为
